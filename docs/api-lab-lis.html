GET /user Obtener información del usuario autenticado

Retorna los datos del usuario actualmente autenticado. Este endpoint requiere que el usuario esté autenticado mediante un token válido. Es comúnmente utilizado para obtener el perfil del usuario en sesión, incluyendo sus datos personales y atributos asociados.

GET /admin Acceder al panel de administración

Endpoint de acceso al panel de administración. Requiere autenticación mediante token Sanctum. Devuelve información o recursos relacionados con el área administrativa de la aplicación. Solo usuarios autenticados con un token válido pueden acceder a este recurso.

Responses

{"message": "This action is unauthorized."}

POST /catalogo Crear un nuevo ítem en el catálogo

Este endpoint permite registrar un nuevo ítem en el catálogo del sistema. Se envían los datos del producto o servicio mediante una solicitud POST. El servidor valida los campos requeridos y, si todo es correcto, persiste el nuevo registro en la base de datos y retorna el ítem creado con su identificador asignado. Si ocurre un error de validación, se retornan los mensajes de error correspondientes.

Parameters
NameTypeRequiredDescription
nombre string required Nombre del ítem del catálogo
descripcion string optional Descripción detallada del ítem del catálogo
precio integer required Precio del ítem del catálogo, debe ser un valor numérico mayor o igual a cero
categoria_id integer required Identificador de la categoría a la que pertenece el ítem; debe existir en la tabla de categorías
imagen file optional Imagen representativa del ítem; debe ser un archivo de imagen válido con tamaño máximo de 2MB
activo boolean optional Indica si el ítem está activo o inactivo en el catálogo; por defecto se considera activo
stock integer optional Cantidad disponible en inventario del ítem del catálogo
Responses

401 No autenticado. El token de autorización no fue proporcionado o es inválido. 
{"message": "Unauthenticated."}
403 No autorizado. El usuario autenticado no tiene permisos para crear ítems en el catálogo. 
{"message": "This action is unauthorized."}
500 Error interno del servidor. Ocurrió un problema inesperado al procesar la solicitud. 
{"message": "Server Error"}

POST /catalogo/{examenId}/estructura Crear un nuevo nodo en la estructura del árbol de catálogo para un examen específico

Se utiliza para definir la organización interna del catálogo, como secciones, categorías o ítems dentro del examen indicado por su identificador. Al crear el nodo, este queda vinculado al examen correspondiente y se integra dentro de la jerarquía existente del catálogo.

descripcion string optional Descripción adicional o detalle del nodo que se está creando en la estructura del catálogo.
activo boolean optional Indica si el nodo estará activo y visible dentro de la estructura del catálogo. Por defecto es true.
