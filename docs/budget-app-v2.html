Categories Controller

5 endpoints
GET /api/categories Obtiene la lista completa de categorías ordenadas alfabéticamente

Retorna todas las categorías registradas en la base de datos, ordenadas de forma ascendente por nombre. La consulta se ejecuta en modo de solo lectura (AsNoTracking) para optimizar el rendimiento. Cada elemento de la lista incluye únicamente el identificador y el nombre de la categoría.

Responses
200 Solicitud exitosa. Retorna un array con todas las categorías disponibles. Si no existen categorías, retorna un array vacío. 
[{"id": 1, "name": "Electrónica"}, {"id": 2, "name": "Ropa"}, {"id": 3, "name": "Hogar"}]
500 Error interno del servidor. Ocurre cuando hay un fallo en la conexión con la base de datos u otro error inesperado durante el procesamiento de la solicitud. 
{"message": "An unexpected error occurred"}
GET /api/categories/{id:int} Obtiene una categoría por su identificador único

Busca y retorna una categoría específica en la base de datos utilizando su identificador entero. La consulta se realiza de forma asíncrona y sin seguimiento de cambios (AsNoTracking) para optimizar el rendimiento. Si no se encuentra ninguna categoría con el identificador proporcionado, se retorna un error 404 Not Found. En caso de éxito, retorna un objeto con el identificador y el nombre de la categoría.

Parameters
NameTypeRequiredDescription
id integer required Identificador único de la categoría a consultar. Debe ser un número entero positivo y estar presente en la URL como parámetro de ruta.
Responses
200 La categoría fue encontrada exitosamente. Retorna el identificador y el nombre de la categoría. 
{"id": 1, "name": "Electrónica"}
404 No se encontró ninguna categoría con el identificador proporcionado. 
{}
POST /api/categories Crear una nueva categoría

Crea una nueva categoría en el sistema a partir del nombre proporcionado. El nombre es recortado de espacios en blanco antes de ser procesado. Si el nombre resultante está vacío, se retorna un error de solicitud incorrecta. Si ya existe una categoría con el mismo nombre (sensible a mayúsculas/minúsculas según la base de datos), se retorna un conflicto. En caso de éxito, la categoría es persistida en la base de datos y se retorna la representación del recurso creado junto con la URL de acceso al mismo.

Parameters
NameTypeRequiredDescription
Name string required Nombre de la categoría a crear. Se aplica trim automáticamente. No puede ser nulo, vacío ni estar compuesto únicamente por espacios en blanco. Debe ser único entre todas las categorías existentes.
Responses
201 Categoría creada exitosamente. Retorna el recurso creado con su identificador y nombre, e incluye el header Location apuntando al endpoint de consulta individual. 
{"id": 1, "name": "Electrónica"}
400 Solicitud incorrecta. Ocurre cuando el campo Name es nulo, está vacío o contiene únicamente espacios en blanco tras aplicar el trim. 
{}
409 Conflicto. Ocurre cuando ya existe una categoría registrada en la base de datos con el mismo nombre. 
{"message": "Ya existe una categoría con ese nombre."}
Request Example
{"name": "Electrónica"}
PUT /api/categories/{id:int} Actualizar una categoría existente por su ID

Actualiza el nombre de una categoría identificada por su ID. El endpoint aplica las siguientes validaciones: el campo 'Name' no puede ser nulo ni vacío (tras aplicar Trim); si la categoría con el ID especificado no existe, retorna 404; si ya existe otra categoría distinta con el mismo nombre, retorna 409 indicando duplicado. Si todas las validaciones pasan, persiste el nuevo nombre y retorna 204 sin contenido.

Parameters
NameTypeRequiredDescription
id integer required Identificador único de la categoría a actualizar. Se pasa como parte de la URL.
Name string required Nuevo nombre que se asignará a la categoría. Se eliminan los espacios en blanco al inicio y al final antes de validar y guardar.
Responses
204 La categoría fue actualizada correctamente. No se retorna contenido en el cuerpo de la respuesta.
400 El campo 'Name' está vacío o contiene solo espacios en blanco.
404 No existe ninguna categoría con el ID proporcionado.
409 Ya existe otra categoría con el mismo nombre. Se retorna un mensaje descriptivo del conflicto. 
{"message": "Ya existe otra categoría con ese nombre."}
Request Example
{"Name": "Electrónica"}
DELETE /api/categories/{id:int} Elimina una categoría existente por su identificador único

Elimina permanentemente una categoría de la base de datos según el ID proporcionado. Antes de proceder con la eliminación, el endpoint verifica que la categoría exista y que no tenga gastos asociados. Si la categoría no existe, retorna 404. Si la categoría tiene uno o más gastos vinculados, retorna 409 indicando que no es posible eliminarla para preservar la integridad referencial de los datos. Si todas las validaciones son exitosas, la categoría es eliminada y se retorna 204 sin contenido.

Parameters
NameTypeRequiredDescription
id integer required Identificador único de la categoría que se desea eliminar. Se pasa como parámetro en la ruta de la URL.
Responses
204 La categoría fue eliminada exitosamente. No se retorna contenido en el cuerpo de la respuesta.
404 No se encontró ninguna categoría con el ID especificado.
409 La categoría no puede ser eliminada porque tiene gastos asociados. Se retorna un mensaje descriptivo del conflicto. 
{"message": "No se puede eliminar la categoría porque tiene gastos asociados."}

GET /api/expenses Obtiene la lista de todos los gastos con filtros opcionales por fecha y categoría

Retorna una colección de gastos registrados en el sistema. Permite filtrar los resultados por fecha de inicio, fecha de fin y/o categoría. Los gastos se devuelven ordenados de forma descendente por fecha y, en caso de empate, por identificador descendente. Cada gasto incluye la información de su categoría asociada si existe. Todos los parámetros de filtro son opcionales; si no se proporcionan, se retornan todos los gastos disponibles.

Parameters
NameTypeRequiredDescription
from string optional Fecha de inicio del rango de búsqueda. Solo se incluirán los gastos cuya fecha sea mayor o igual a este valor.
to string optional Fecha de fin del rango de búsqueda. Solo se incluirán los gastos cuya fecha sea menor o igual a este valor.
categoryId integer optional Identificador único de la categoría. Filtra los gastos que pertenecen exclusivamente a esta categoría.
Responses
200 Solicitud exitosa. Retorna un arreglo con los gastos que coinciden con los filtros aplicados. Si no hay gastos, retorna un arreglo vacío. 
[{"id": 42, "amount": 150.75, "date": "2024-06-15", "description": "Compra de suministros de oficina", "categoryId": 3, "categoryName": "Oficina"}, {"id": 41, "amount": 60.00, "date": "2024-06-14", "description": "Almuerzo con cliente", "categoryId": 5, "categoryName": "Alimentación"}, {"id": 40, "amount": 200.00, "date": "2024-06-10", "description": "Transporte", "categoryId": null, "categoryName": null}]
GET /api/expenses/{id:int} Obtiene un gasto específico por su identificador único

Recupera los detalles completos de un gasto a partir de su ID entero. El endpoint realiza una consulta a la base de datos incluyendo la categoría asociada (eager loading). Si no se encuentra ningún gasto con el ID proporcionado, se retorna un error 404. La consulta se ejecuta en modo de solo lectura (AsNoTracking) para optimizar el rendimiento.

Parameters
NameTypeRequiredDescription
id integer required Identificador único del gasto a recuperar. Debe ser un número entero positivo y corresponder a un gasto existente en la base de datos.
Responses
200 El gasto fue encontrado exitosamente. Retorna el objeto completo del gasto incluyendo información de su categoría asociada. 
{"id": 42, "amount": 150.75, "date": "2024-06-15T10:30:00", "description": "Compra de material de oficina", "categoryId": 3, "categoryName": "Oficina"}
404 No existe ningún gasto con el ID proporcionado en la base de datos. 
{}
POST /api/expenses Crea un nuevo gasto en el sistema

Registra un nuevo gasto en la base de datos con los datos proporcionados. Valida que el importe sea mayor que cero y que la categoría especificada exista previamente en el sistema. Si las validaciones son correctas, persiste el gasto, carga la información de la categoría asociada y retorna el recurso creado junto con la URL de acceso al mismo.

Date string required Fecha en la que se realizó el gasto.
Description string required Descripción o concepto del gasto. Los espacios al inicio y al final serán eliminados automáticamente antes de guardarse.
CategoryId integer required Identificador de la categoría a la que pertenece el gasto. Debe corresponder a una categoría existente en el sistema.
Responses
201 El gasto fue creado exitosamente. Se retorna el recurso creado junto con el header Location apuntando al endpoint de consulta individual.

400 La solicitud es inválida. Puede deberse a que el importe es menor o igual a cero, o a que la categoría especificada no existe en el sistema. 
{"message": "El importe debe ser mayor que cero."}
400 La categoría proporcionada no existe en la base de datos. 
{"message": "La categoría no existe."}

{"type": "https://tools.ietf.org/html/rfc7231#section-6.5.1", "title": "One or more validation errors occurred.", "status": 422, "errors": {"Amount": ["The Amount field is required."]}}

PUT /api/expenses/{id:int} Actualiza un gasto existente por su identificador

Permite modificar los datos de un gasto registrado en el sistema. Se valida que el importe sea mayor que cero, que la categoría especificada exista en la base de datos y que el gasto identificado por el parámetro de ruta exista. Si todas las validaciones son exitosas, se actualizan los campos amount, date, description (aplicando trim) y categoryId del registro, y se persisten los cambios. Retorna 204 No Content en caso de éxito, 400 Bad Request si los datos son inválidos o la categoría no existe, y 404 Not Found si el gasto no existe.

DELETE /api/expenses/{id:int} Elimina un gasto existente por su identificador único

Este endpoint elimina permanentemente un registro de gasto de la base de datos. Se busca el gasto por su identificador entero; si no existe, se retorna un error 404. Si el gasto es encontrado, se elimina de forma definitiva e irreversible y se confirman los cambios en la base de datos. No se aplica borrado lógico (soft delete), por lo que la operación no puede deshacerse.

Parameters
NameTypeRequiredDescription
id integer required Identificador único del gasto que se desea eliminar. Debe ser un número entero válido correspondiente a un gasto existente en la base de datos.
Responses
204 El gasto fue eliminado exitosamente. No se retorna contenido en el cuerpo de la respuesta.
404 No se encontró ningún gasto con el identificador proporcionado. El recurso no existe en la base de datos. 
{"status": 404, "message": "Not Found"}
