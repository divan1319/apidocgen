GET /api/projects Obtiene la lista de todos los proyectos disponibles

Este endpoint retorna una colección de todos los proyectos registrados en el sistema. Realiza una consulta general sin filtros obligatorios, devolviendo todos los registros accesibles. Puede ser utilizado para poblar listados, selectores o paneles de administración. Dependiendo de la implementación, puede incluir paginación o retornar todos los proyectos en una sola respuesta.

GET /api/projects/{slug} Obtiene los detalles de un proyecto específico por su slug

Recupera la información completa de un proyecto identificado de forma única por su slug en la URL. El slug es un identificador legible que representa al proyecto. Si el proyecto no existe, se retorna un error 404. Este endpoint es de solo lectura y no modifica ningún recurso.

Parameters
NameTypeRequiredDescription
slug string required Identificador único legible del proyecto (slug) incluido en la ruta de la URL. Se utiliza para localizar el proyecto en el sistema.
Responses
200 El proyecto fue encontrado y se retorna su información completa. 
{"id": 1, "slug": "mi-proyecto", "name": "Mi Proyecto", "description": "Descripción del proyecto", "created_at": "2024-01-01T00:00:00Z", "updated_at": "2024-01-01T00:00:00Z"}
404 No se encontró ningún proyecto con el slug proporcionado. 
{"error": "project not found"}
500 Error interno del servidor al intentar recuperar el proyecto. 
{"error": "internal server error"}
POST /api/projects Crear un nuevo proyecto

Este endpoint permite crear un nuevo proyecto en el sistema. Se espera que el cliente envíe los datos del proyecto en el cuerpo de la solicitud en formato JSON. Al completarse exitosamente, el servidor retorna el proyecto recién creado con su identificador único asignado. El endpoint está registrado en un multiplexor HTTP estándar de Go.

Parameters

name string required Nombre del proyecto a crear
description string optional Descripción detallada del proyecto
status string optional Estado inicial del proyecto. Valores permitidos: active, inactive, draft
owner_id integer optional Identificador del usuario propietario del proyecto
Responses
201 Proyecto creado exitosamente. Retorna el objeto del proyecto con su identificador único asignado. 
{"id": 1, "name": "Mi Proyecto", "description": "Descripción del proyecto", "status": "active", "owner_id": 42, "created_at": "2024-01-15T10:30:00Z", "updated_at": "2024-01-15T10:30:00Z"}
400 Solicitud inválida. El cuerpo de la solicitud contiene datos malformados o campos requeridos faltantes. 
{"error": "bad request", "message": "invalid request body"}
422 Error de validación. Uno o más campos no cumplen con las reglas de validación establecidas. 
{"error": "unprocessable entity", "message": "validation failed", "details": {"name": "field is required"}}
500 Error interno del servidor. Ocurrió un problema inesperado al intentar crear el proyecto. 
{"error": "internal server error", "message": "an unexpected error occurred"}
Request Example
{"name": "Mi Nuevo Proyecto", "description": "Este es un proyecto de ejemplo", "status": "active", "owner_id": 42}

ALL /docs/ Sirve la documentación estática de la API desde el directorio /docs/

Este endpoint sirve archivos estáticos de documentación accesibles bajo la ruta /docs/. Utiliza el middleware http.StripPrefix de Go para eliminar el prefijo /docs/ de la ruta antes de buscar el archivo en el sistema de archivos o servidor de archivos configurado. Acepta todos los métodos HTTP estándar (GET, HEAD, etc.), aunque en la práctica solo GET y HEAD son relevantes para servir contenido estático. No requiere autenticación en la mayoría de configuraciones. El comportamiento exacto depende del file server configurado detrás del StripPrefix.

Parameters
NameTypeRequiredDescription
path string optional Ruta relativa al recurso de documentación deseado dentro del directorio /docs/. Por ejemplo, /docs/index.html accedería al archivo index.html. Si se omite, generalmente se sirve el archivo index por defecto.
Responses
200 El archivo solicitado fue encontrado y se retorna correctamente. El Content-Type varía según la extensión del archivo (text/html, application/json, text/css, etc.). 
"<contenido del archivo estático solicitado>"
301 Redirección permanente, generalmente ocurre cuando se accede a un directorio sin barra final. 
{"Location": "/docs/"}
404 El archivo o recurso solicitado no fue encontrado en el directorio de documentación. 
"404 page not found"
403 Acceso prohibido al recurso solicitado, por ejemplo al intentar listar un directorio sin archivo index. 
"403 Forbidden"
405 Método HTTP no permitido. Solo GET y HEAD son soportados para archivos estáticos. 
"405 Method Not Allowed"
ALL / Manejador principal de la aplicación de página única (SPA)

Este endpoint actúa como el punto de entrada principal para una aplicación de página única (SPA). Responde a todos los métodos HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD) en la ruta raíz '/'. El manejador spaHandler se encarga de servir los archivos estáticos de la SPA y redirigir todas las rutas no reconocidas al archivo index.html, permitiendo que el enrutamiento del lado del cliente tome el control. Este patrón es común en aplicaciones construidas con frameworks como React, Vue o Angular.

Parameters
NameTypeRequiredDescription
path string optional Cualquier segmento de ruta adicional será manejado por el enrutador del lado del cliente de la SPA. No se requieren parámetros específicos para este endpoint.
Responses
200 Respuesta exitosa. Devuelve el archivo estático solicitado o el archivo index.html de la SPA para que el enrutador del cliente gestione la navegación. 
<!DOCTYPE html><html lang="en"><head><meta charset="UTF-8"><title>SPA App</title></head><body><div id="root"></div></body></html>
301 Redirección permanente. Puede ocurrir si se accede a la ruta sin el slash final u otras redirecciones canónicas. 
<a href="/">Moved Permanently</a>
304 No modificado. El archivo estático solicitado no ha cambiado desde la última solicitud, se utilizan los recursos en caché del cliente.
404 No encontrado. En algunas configuraciones de spaHandler, si el archivo estático no existe y no hay fallback a index.html configurado, se devuelve este código. 
404 page not found
500 Error interno del servidor. Ocurre si el spaHandler no puede leer o servir los archivos estáticos requeridos. 
500 Internal Server Error
