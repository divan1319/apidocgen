Categories Controller

5 endpoints
GET /api/categories Retrieve all categories ordered by name

Returns a complete list of all categories available in the system, sorted alphabetically by name. This is a read-only operation that does not modify any data. Results are fetched without change tracking for optimal performance.

Responses
200 A sorted list of all categories was successfully retrieved. Returns an empty array if no categories exist. 
[{"id": 1, "name": "Electronics"}, {"id": 2, "name": "Furniture"}, {"id": 3, "name": "Clothing"}]
GET /api/categories/{id:int} Retrieve a category by its ID

Fetches a single category record from the database matching the provided integer ID. The query is executed as a read-only (non-tracking) operation. If no category is found with the given ID, a 404 Not Found response is returned. On success, the category's ID and name are returned in the response body.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the category to retrieve. Must be an integer and is supplied as part of the URL path.
Responses
200 The category was found and returned successfully. 
{"id": 1, "name": "Electronics"}
404 No category with the specified ID exists in the database. 
{}
POST /api/categories Create a new category

Creates a new category with the provided name. The name is trimmed of leading and trailing whitespace before processing. Returns 400 Bad Request if the trimmed name is empty, 409 Conflict if a category with the same name already exists, or 201 Created with the new category data upon success.

Parameters
NameTypeRequiredDescription
Name string required The name of the category to create. Will be trimmed of whitespace. Must not be empty after trimming and must be unique among existing categories.
Responses
201 Category created successfully. The response includes the newly created category's ID and name, and a Location header pointing to the new resource. 
{"id": 1, "name": "Electronics"}
400 Bad Request. Returned when the provided name is null, empty, or contains only whitespace after trimming. 
{}
409 Conflict. Returned when a category with the same name already exists in the database. 
{"message": "Ya existe una categoría con ese nombre."}
Request Example
{"name": "Electronics"}
PUT /api/categories/{id:int} Update an existing category by ID

Updates the name of an existing category identified by its integer ID. The provided name is trimmed of whitespace before processing. Returns 400 if the name is empty after trimming, 404 if no category with the given ID exists, 409 if another category already uses the same name, or 204 on successful update.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the category to update.
Name string required The new name for the category. Leading and trailing whitespace will be trimmed automatically. Must be unique across all other categories.
Responses
204 Category was successfully updated. No content is returned.
400 The request body was invalid, typically because the Name field is missing, null, or results in an empty string after trimming.
404 No category with the specified ID was found in the database.
409 Another category (with a different ID) already has the same name, causing a conflict. 
{"message": "Ya existe otra categoría con ese nombre."}
Request Example
{"Name": "Electronics"}
DELETE /api/categories/{id:int} Delete a category by ID

Permanently deletes the category with the specified ID from the database. Before deletion, the endpoint verifies that the category exists and that it has no associated expenses. If the category does not exist, a 404 is returned. If the category has one or more linked expenses, the deletion is rejected with a 409 Conflict to preserve referential integrity. On success, no content is returned.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the category to delete. Must be a valid integer corresponding to an existing category.
Responses
204 The category was successfully deleted. No content is returned.
404 No category was found with the provided ID.
409 The category cannot be deleted because it has one or more associated expenses. The relationship must be removed before the category can be deleted. 
{"message": "No se puede eliminar la categoría porque tiene gastos asociados."}

GET /api/expenses Retrieve a list of all expenses with optional filtering

Returns a list of expense records from the database, optionally filtered by date range and/or category. Results are ordered by date descending, then by ID descending (most recent first). Each expense includes its associated category name if a category is linked. All query parameters are optional and can be combined freely.

Parameters
NameTypeRequiredDescription
from string optional Filter expenses with a date greater than or equal to this value (inclusive lower bound of the date range).
to string optional Filter expenses with a date less than or equal to this value (inclusive upper bound of the date range).
categoryId integer optional Filter expenses belonging to the specified category ID. Only expenses whose categoryId matches this value will be returned.
Responses
200 Successfully retrieved the list of expenses matching the provided filters. Returns an empty array if no expenses match. 
[{"id": 42, "amount": 149.99, "date": "2024-06-15", "description": "Office supplies", "categoryId": 3, "categoryName": "Office"}]
GET /api/expenses/{id:int} Retrieve a single expense by its ID

Fetches a single expense record from the database by its integer ID. The expense is returned along with its associated category name. If no expense exists with the given ID, a 404 Not Found response is returned. The query is read-only (no tracking) and eagerly loads the related Category entity.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the expense to retrieve. Must be a positive integer and is provided as part of the URL path.
Responses
200 The expense record was found and is returned successfully, including its associated category information. 
{"id": 42, "amount": 150.75, "date": "2024-06-15T00:00:00Z", "description": "Office supplies", "categoryId": 3, "categoryName": "Office"}
404 No expense with the specified ID was found in the database. 
{"title": "Not Found", "status": 404}
POST /api/expenses Create a new expense record

Creates a new expense entry in the database. Validates that the amount is greater than zero and that the referenced category exists before persisting. On success, loads the associated category and returns the full expense representation with a 201 Created status pointing to the newly created resource. Returns 400 Bad Request if the amount is non-positive or the category does not exist, and 422 Unprocessable Entity if the request body fails model validation.

Parameters
NameTypeRequiredDescription
Amount number required The monetary value of the expense. Must be a positive number greater than zero.
Date string required The date on which the expense occurred, in ISO 8601 format (e.g. 2024-06-15 or 2024-06-15T00:00:00Z).
Description string required A textual description of the expense. Leading and trailing whitespace is trimmed before storage.
CategoryId integer required The unique identifier of the category to which this expense belongs. The category must already exist in the database.
Responses
201 Expense created successfully. Returns the full expense object including the resolved category name. The Location header points to the newly created resource. 
{"id": 42, "amount": 125.50, "date": "2024-06-15T00:00:00Z", "description": "Team lunch", "categoryId": 3, "categoryName": "Food & Beverage"}
400 Bad request due to a business rule violation: the amount is less than or equal to zero, or the specified category does not exist. 
{"message": "El importe debe ser mayor que cero."}
422 Model validation failed. One or more required fields are missing or have an invalid format. 
{"type": "https://tools.ietf.org/html/rfc4918#section-11.2", "title": "One or more validation errors occurred.", "status": 422, "errors": {"Amount": ["The Amount field is required."]}}
Request Example
{"amount": 125.50, "date": "2024-06-15T00:00:00Z", "description": "Team lunch", "categoryId": 3}
PUT /api/expenses/{id:int} Update an existing expense by ID

Updates the fields of an existing expense record identified by its integer ID. The request body must contain all required fields: Amount, Date, Description, and CategoryId. Validation is enforced at the model level and additionally via business rules: Amount must be greater than zero, and the referenced CategoryId must exist in the database. If the expense with the given ID does not exist, a 404 is returned. On success, no content is returned (204). Description is trimmed of leading and trailing whitespace before being saved.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the expense to update.
Amount integer required The monetary amount of the expense. Must be a positive value greater than zero.
Date string required The date on which the expense occurred, in ISO 8601 format (e.g., YYYY-MM-DD).
Description string required A textual description of the expense. Leading and trailing whitespace will be trimmed before saving.
CategoryId integer required The ID of the category to associate with this expense. Must reference an existing category record.
Responses
204 The expense was successfully updated. No content is returned in the response body.
400 Bad request due to validation failure, Amount being zero or negative, or the specified CategoryId not existing. 
{"message": "El importe debe ser mayor que cero."} or {"message": "La categoría no existe."} or {"errors": {"Amount": ["The Amount field is required."]}, "title": "One or more validation errors occurred.", "status": 400}
404 No expense was found matching the provided ID. 
{"status": 404, "title": "Not Found"}
Request Example
{"Amount": 150.00, "Date": "2024-06-15", "Description": "Office supplies purchase", "CategoryId": 3}
DELETE /api/expenses/{id:int} Delete an expense by ID

Permanently deletes the expense record identified by the given integer ID. If no expense with the specified ID exists, a 404 Not Found response is returned. On successful deletion, the record is removed from the database and a 204 No Content response is returned with no response body.

Parameters
NameTypeRequiredDescription
id integer required The unique identifier of the expense to delete. Must be a positive integer and is provided as a route parameter.
Responses
204 The expense was successfully found and permanently deleted. No content is returned in the response body.
404 No expense with the specified ID was found in the database. The deletion was not performed. 
{"status": 404, "message": "Not Found"}
